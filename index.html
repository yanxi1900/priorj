Welcome to PriorJ Project!

ProrJ is a tool which has as goal allow the automation of the main prioritization techniques for the Java/JUnit world. As secondary features PriorJ also give to the tester/developer artifacts which can guide the choosing of the best prioritization technique according to the development characteristics.

In summary, PriorJ has the following functionalities:

  • Instrumentation and test coverage analysis for Java source code;
  • Automation of the main prioritization techniques using the JUnit framework;
  • Generation of automatic mechanisms for execution of prioritized suites;
  • Selection of test cases according with restrictions defined by the user;
  • An environment for evaluation/comparison of prioritization techniques;
  • Storage and recovery of historical data.
  • Plugin for Eclipse IDE

Prioritization Techniques

Until the present moment the PriorJ tool implements and deals with six prioritization techniques. In the sequence, we briefly present them and their prioritization processes:

  • Total Statement Coverage (TSC): This technique schedules test cases based on the number of statements that each one covers.
  • Total Method Coverage (TMC): Similar to the TSC, differing by using the number of covered methods instead of covered statements as ordering criteria.
  • Additional Statement Coverage (ASC): The first test case is the one which covers more statements. The next test case to be chosen is the one which covers more statements that were not covered yet by the previously chosen test cases. This process is repeated until the prioritized test suite is complete.
  • Additional Method Coverage (AMC): Technique similar to the AMC, differing from ASC by using the coverage of methods as ordering criteria, instead of statement coverage.
  • Random (RD): Technique in which the test cases order in the prioritized test suite is defined randomly.
  • Changed Blocks (proposed by Srivastava and Thiagarajan): This technique firstly identifies the changed blocks between the two versions (by binary analysis) and schedule the test cases according to how much of the changed block each test covers.

Related Modules

This project consists of other projects, divided into modules.

  • Coverage - PriorJ API for Coverage.
  • Core - Module to check for differences between versions of the same project.
  • JavaIO - Module for Input/Output Services.
  • Eclipse Plugin - Plugin to allow use of the tool within the Eclipse IDE.
  • Report - Module for generating reports and APFD charts using JavaScript and Bootstrap.
